## 0. ⛔ ОБОВʼЯЗКОВЕ: без AI-дефолтів, рівень Awwwards

Дизайн-система нижче — це **технічний каркас під Webflow**, а не візуальний рецепт. Візуально кожен проєкт
має бути **унікальним і креативним, як на Awwwards**. Активуй скіл **`frontend-design`** на кожен новий UI.

**НЕ додавай за замовчуванням типові AI-елементи:**
- ❌ **Eyebrow-сабтайтл з крапкою** (`• LABEL`) над секціями — **тільки якщо дизайнер прямо попросив**.
- ❌ Номери `01/02/03`, якщо контент не є справжньою послідовністю.
- ❌ Шаблонний hero «велика цифра + лейбл + градієнт».
- ❌ Стокові AI-палітри (кремовий+serif+теракота; чорний фон+кислотний акцент; «газетний» hairline-лейаут) — якщо цього не вимагає бриф.
- ❌ Зайва анімація всюди, симетричні блоки-близнюки, декор без сенсу.

**Натомість:** hero = теза (найхарактерніше в світі продукту); типографіка несе характер; структура (eyebrow/номери/лейбли)
кодує зміст, а не декорує; один сміливий **signature-елемент**, решта тиха; свідомий ризик; quality floor завжди
(адаптив, видимий keyboard-focus, `prefers-reduced-motion`). Перед версткою склади короткий план (палітра, шрифти,
лейаут, signature) і звір з брифом: якщо частина = дефолт для будь-якої схожої сторінки — переробити.

## 1. Налаштування середовища

**Де запускати.** Для дизайну з локальною папкою — **Claude Code у терміналі/десктопі** (веб працює лише з GitHub-репо в хмарі):
```bash
npm install -g @anthropic-ai/claude-code
mkdir ~/Desktop/new-project && cd ~/Desktop/new-project && claude
```

**Конектори (MCP)** — `/mcp` → авторизація:
- **Webflow** (обовʼязково) — пуш елементів/класів/CMS у Designer.
- **Vercel** (бажано) — превʼю-лінк до Webflow. **GitHub** (опц.) — версіонування. **Higgsfield** — генерація зображень.

**Скіли** ставимо **глобально** (працюють у всіх проєктах, ставляться раз на машині — не в git проєкту):
```bash
npm i -g ui-ux-pro-max-cli && uipro init --ai claude --global   # → ~/.claude/skills/
# frontend-design → скопіювати в ~/.claude/skills/frontend-design/
```
Корисні скіли: `ui-ux-pro-max` (стилі/палітри/шрифти/чарти), `frontend-design` (унікальний дизайн, розділ 0),
`webflow-converter`, `webflow-cms-collections`, `component-sync` (компоненти, 2.11), `design-memory`
(контекст між чатами), `design-prompt-generator`, `image-sourcing`.

**Головне:** поклади `CLAUDE.md` у корінь проєкту з правилами розділу 2 — Claude читає його на старті й тримає правила автоматично.

## 2. Дизайн-система (постав у `CLAUDE.md`)

### 2.1 Брейкпоінти (Webflow → Project Settings → Breakpoints, ДО верстки)

| Брейкпоінт | Діапазон | Напрям |
|---|---|---|
| **Base** (default) | 992–1279 | базові стилі, каскад ↑↓ |
| Large / XL / XXL | 1280–1439 / 1440–1919 / 1920+ | ↑ up |
| Medium / Small / Tiny | 768–991 / 568–767 / 360–567 | ↓ down |

Стилі з Base каскадяться і вгору, і вниз. **Оверрайд — лише де значення реально відрізняється від Base.**
Перевіряти всі точки: **1920 / 1440 / 1280 / 992 / 768 / 568 / 360**.

### 2.2 Структура секції (однакова скрізь)

```html
<section class="section section-name">
  <div class="base-container">
    <div class="section-head">
      <!-- eyebrow — ТІЛЬКИ за проханням дизайнера (розділ 0), не за замовчуванням -->
      <h2 class="heading-spacer-top">Section Title</h2>
      <p class="lead">Supporting description.</p>
    </div>
    <!-- section content -->
  </div>
</section>
```

- Секція = `<section>` + `section` + сабклас (`section trust`). Контент — у `base-container`.
- **`base-container`:** `max-width` задає дизайнер під проєкт (у змінну, однакову для всіх), `margin: auto`, `width: 100%`, `padding: 0 15px`.
- **Падінги секції — тільки через `section` / `section-top` / `section-bot`** (шкала нижче), не на власному класі секції.
- Темні секції: фон на класі секції; діти отримують `on-dark`. Статичні секції (CTA, Footer) — ніколи не в CMS.

**Section Padding (перший клас секції):**

| Клас | Base | 1280+ | 768–991 | ≤767 |
|---|---|---|---|---|
| `section` | 100px 0 | 120px 0 | 80px 0 | 60px 0 |
| `section-top` | 100px 0 0 | 120px 0 0 | 80px 0 0 | 60px 0 0 |
| `section-bot` | 0 0 100px | 0 0 120px | 0 0 80px | 0 0 60px |

### 2.3 Нейминг класів

- **lowercase-hyphen**, секційний префікс: `trust-head`, `approach-title`, `foot-grid`.
- **Макс 2 класи**: структурний + модифікатор (`btn btn-primary`, `eyebrow on-dark`).
- Модифікатори без префікса (`on-dark`, `on-light`, `high`); стани — `is-*`; CMS-маркери — `field-*` (знімаються після біндингу).
- **Кожен елемент має власний клас — жодних descendant-селекторів** (✅ `.approach-title` ❌ `.trust-head h2`). Без інлайн-стилів.

Ключові утиліти: `on-dark`/`on-light` (світлі/темні тони), `base-container`, `heading-spacer-top` (mt 22px),
`small-spacer-top` (mt 10px), `link-reset`, `relative` (z-index стек), `high` (акцент у заголовку).

### 2.4 Кольори — Color Variables

**Значення різні в кожному проєкті** (приклад нижче — плейсхолдер). Незмінне — **система**:
правильні назви ролей (Accent, Ink, Muted, Line Light…), уніфікація (один колір = одна змінна),
**перевірка контрасту WCAG** (≥4.5:1 body, ≥3:1 великі заголовки/UI) на реальних поєднаннях.

Ролі-приклад: `Accent #0162ff` · `Accent Light #eaf1ff` · `Accent Dark #0148bf` · `White #fff` ·
`Off White #FAFAF9` · `Ink #111` · `Muted rgba(17,17,17,.62)` · `Line Light rgba(17,17,17,.10)` · `Line Dark rgba(255,255,255,.16)`.

### 2.5 Типографіка — Font Variables

**Шрифти обирає дизайнер під проєкт.** Тверді правила:
- **Максимум 2 шрифти**; **більше 3 — НІКОЛИ не допускай** (натяк на 4-й → стоп, перепитай).
- **Бажано Google Fonts.** Не з Google Fonts → спершу **перевір ліцензію на web-embed або перепитай**, не став самовільно.
- Ролі: **Primary → заголовки H1–H6, лого, декор**; **Secondary → body/параграфи, UI, навігація, футер**.
- **Кнопки/лінки** — вибір дизайнера; **дефолт: Secondary** (читабельніший на малому й для інтерактиву).

Шкали (приклад): body XXS 14 → XXL 24px; weights 300–800; line-height 1em→1.55; letter-spacing −1.5→0px.
Заголовкові розміри — Variables з режимами по брейкпоінтах (оверрайд де відрізняється від Base).

### 2.6 Spacing / Radius
Spacing — Variables з режимами по брейкпоінтах; текстовий margin → змінна **Spacing None (=0)**.
Radius: XS 4 (теги) · S 6 (інпути) · M 8 (картки) · L 10 (панелі) · XL 100px (пігулки — кнопки/бейджі).

### 2.7 Кнопки, лінки, розмітка під конвертер

**Набір кнопок/лінків визначає дизайнер під кожен UI** — незмінна лише система:
- База `.btn` + один варіант, макс 2 класи; hover/стани — у Webflow States panel.
- **Кнопки → завжди `<a>`** (не `<button>`); іконка+текст — окремими прямими дітьми; тоглам `e.preventDefault()`.
- **Лінки — ніколи не голий `<a>`**: клас під призначення (`more`, `explore`, `link-reset`, `card-link`, `foot-link`).
- **Інпути → `<div>`-плейсхолдер**; **форми → `<div class="signup-form">`** (не `<form>`); реальні Input/Form — у Designer.
- Кожен декоративний `<svg>` має **`data-icon="…"`**. Без `<br>`.
- **Усі гепи/спейси — через flex/grid-контейнери** (`grid-row-gap`/`grid-column-gap` на батьку), **а не** margin/padding
  на самих текстах/лінках/`<p>`. Текст сам відступів не несе — відступ на обгортці (`*-wrap`/`section-head`).

### 2.8 ⚠️ БЕЗ Custom properties (головне правило експорту)
Після пушу панель **Custom properties має бути ПОРОЖНЬОЮ**. Тому:
- **CSS тільки longhand** (сторони border/padding/margin окремо, 4 радіуси окремо, `background-color`/`background-image` окремо).
- **`grid-row-gap`/`grid-column-gap`, ніколи `gap`.**
- **Градієнтні/сіткові фони** (`linear-gradient` + `background-size`) — у **Site Head Code**, не на класі.
- Жодних кастомних властивостей на класах.

### 2.9 Анімації — тільки атрибути (IX2/GSAP підхоплює)
- `data-load="fade-in"` — над згином (hero, navbar), на завантаженні.
- `data-scroll="fade-in"` — кожна секція на скролі.
- `data-scroll="fade-in-item"` — контент-дівблоки всередині (картки, колонки, заголовковий wrap; сітки — покартково/стаггер).
  Пропускати: секції, контейнери, чистий декор, «голі» заголовки, глибокі вузли.

### 2.10 ⛔ ОБОВʼЯЗКОВЕ: Компоненти = один майстер-елемент
Якщо дизайнер називає елемент **Компонентом** (navbar, footer, картка, кнопка, CTA…) — це **ОДИН майстер**, ідентичний скрізь:
- Оголошуємо майстром через **`component-sync`** і реєструємо **всі** місця, де він є.
- **Правка в одному місці → автоматично в усі інстанси** на всіх сторінках. Ручного дублювання по сторінках немає.
- Ідентичність повна (розмітка/класи/контент/стилі/стани); відрізняється лише свідомо винесене в проп/варіант.
- **Перед будь-якою правкою елемента перевір, чи він не компонент** — якщо так, правимо через `component-sync`.
- У Webflow це **Symbols/Components** (навбар/футер лишаємо як Symbol).

## 3. Процес побудови сайту

0. **Фундамент:** папка + `CLAUDE.md` (розділ 2); конектори + скіли; у Webflow — 7 брейкпоінтів і Variables (кольори/шрифти/spacing/radius).
1. **Style Guide сторінка спершу** — кольори, типо-шкала, кнопки, лінки, картки живими елементами з правильними класами (тест, що система зайшла).
2. **Компоненти** (правило 2.10): збери navbar/footer/картки/CTA раз → `component-sync` → реєстрація всіх інстансів. Навбар/дропдауни — Symbol у Designer вручну.
3. **Сторінки секція-за-секцією** — працюй з **кодом**, не зі скріншотами; шаблон 2.2; нейминг 2.3. Точкова правка = міняй лише те, що просили. Зображення — `image-sourcing`.
4. **Респонсив** — усі 7 точок; оверрайд де відрізняється від Base; console = 0 помилок.
5. **CMS** — динамічні елементи = `field-[slug]`; статичні (CTA, Explore more, Footer, Nav) не біндимо. Колекції — `webflow-cms-collections`.
6. **Анімації** — атрибути 2.9.
7. **Handoff/превʼю** — (опц.) Vercel-лінк; Handoff-консоль (Copy CSS/HTML посекційно), регенерувати завжди з conversion-файлу.

**Приклад індексу секцій:** Hero `hero on-dark` · Trust `section trust` · Approach `section approach on-dark` ·
Platform `section-top platform` · Products/Industries `section-top …` · Testimonials `section testimonials` ·
Final CTA+Footer `cta-footer on-dark` (div-wrapper, не `<section>`).

<a id="4"></a>
## 4. Експорт у Webflow (розширення-білдер)

CMS-флоу (Step 4), по порядку: **Annotate** (`field-[slug]`) → **Build+Apply** → назва колекції+slug →
**Scan Fields** → типи полів (Plain/Rich/Image) → **Create Schema** → **Load Items** → **Push Items** →
**Bind Fields** → **Remove Markers**.

**API НЕ відновлює** (робимо в Designer вручну): CMS List → Source; мегаменю/дропдауни; градієнтні фони (→ Site Head Code);
реальні Input/Form. При пуші змінні біндяться: кольори по значенню, розміри ±1px з розділенням груп (font-size→Typography, spacing→Spacing).
